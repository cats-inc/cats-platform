# A2A Templates

> Starter templates for Agent-to-Agent (A2A) interoperability.

## Contents

- `agent-card.json.example` / `agent-card.yaml.example`: Declare agent identity and capabilities.
- `task.json.example` / `task.yaml.example`: Define a common task payload schema.

## Usage

1. Choose JSON or YAML and copy to non-example files if needed.
2. Align fields with the A2A spec version you implement.
3. Keep `AGENTS.md` and agent-specific files in sync with the Agent Card.
4. Do not store secrets in these files.
